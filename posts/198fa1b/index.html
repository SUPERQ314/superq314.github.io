数据类型判断方法

发表于2022-08-01|更新于2022-09-30
|字数总计:746|阅读时长:3分钟

可用于判断基本数据类型

对于原始数据类型来说，除了null都可以显示正确的类型

  • 对于null，返回object
  • 除了function以外的引用类型，都会返回object

可用于判断引用数据类型
instanceof实现原理

内部机制是测试构造函数的 prototype 属性是否出现在对象原型链中的任何位置。

实现 instanceof：

  1. 首先获取类型的原型
  2. 然后获得对象的原型
  3. 然后一直循环判断对象的原型是否等于类型的原型，直到对象原型为 null，因为原型链最终为 null

constructor在某些场景下失效

创建了一个对象，并更改它的原型，constructor就无法做出准确的判断

constructor 和 instanceof 作为两种检测数据类型的方法，仍然存在一些漏洞

由于每一个 frame都有自己的一套执行环境，跨 frame实例化的对象彼此并不共享原型链，通过 instanceof操作符和 constructor属性检测的方法会失败。
instanceof和 constructor不可跨 frame
需要在同个 frame中才有效。

使用 Object 对象的原型方法 toString ，使用 call 进行狸猫换太子

只检测特定的属性，如果存在，则认为是该类型
