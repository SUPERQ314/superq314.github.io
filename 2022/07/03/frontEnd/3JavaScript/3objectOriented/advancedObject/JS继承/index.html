this、call、apply、bind

详情可看：「前端料包」一文彻底搞懂JavaScript中的this、call、apply和bind

  1. 在浏览器里，在全局范围内this指向window对象
  2. 在函数中，this永远指向最后调用他的那个对象
  3. 构造函数中，this指向new出来的那个新的对象
  4. call、apply、bind中的this被强绑定在指定的那个对象上
  5. 箭头函数中this比较特殊，箭头函数this为父作用域的this，不是调用时的this。要知道前四种方式，都是调用时确定，也就是动态的，而箭头函数的this指向是静态的，声明的时候就确定了下来
  6. apply、call、bind都是js给函数内置的一些API，调用他们可以为函数指定this的执行，同时也可以传参

JavaScript原型及原型链

重温一下构造函数、原型和实例的关系：

每个构造函数都有一个原型对象，原型有一个属性指回构造函数，而实例有一个内部指针指向原型。
如下图：

  • Person 就是一个构造函数
  • Person Prototype 就是原型对象
  • person1, person2就是两个实例

如果原型是另一个类型的实例呢？那就意味着这个原型本身有一个内部指针指向另一个原型，相应地另一个原型也有一个指针指向另一个构造函数。这样就在实例和原型之间构造了一条原型链。这就是原型链的基本构想。

在js中我们是使用构造函数来新建一个对象的，每一个构造函数的内部都有一个prototype属性值，这个属性值是一个对象，这个对象包含了可以由该构造函数的所有实例共享的属性和方法。
当我们使用构造函数新建一个对象后，在这个对象的内部将包含一个指针proto，这个指针指向构造函数的prototype属性对应的值，在ES5中这个指针被称为对象的原型。一般来说我们是不应该能够获取到这个值的，但是现在浏览器中都实现了proto属性来让我们访问这个属性，但是我们最好不要使用这个属性，因为它不是规范中规定的。ES5中新增了一个Object.getPrototypeOf()方法，我们可以通过这个方法来获取对象的原型。

当我们访问一个对象的属性时，如果这个对象内部不存在这个属性，那么它就会去它的原型对象里找这个属性，这个原型对象又会有自己的原型，于是就这样一直找下去，也就是原型链的概念。原型链的尽头一般来说都是Object.prototype所以这就是我们新建的对象为什么能够使用toString()等方法的原因。

原型链:每个对象有一个 __proto__属性，表示自己的原型，每个对象的原型 都指向 该对象的原型对象 prototype，而每个对象的 __proto__.constructor 都指向该对象构造函数的实例，当我们查找的时候，会先沿着自身的属性去查找，找不到去原型上去查找，如果依然没有，则前往该对象上的原型对象进行查找，最终返回结果，如果没有，则返回 null

特点：
JavaScript 对象是通过引用来传递的，我们创建的每个新对象实体中并没有一份属于自己的原型副本。当我们修改原型时，与 之相关的对象也会继承这一改变。

参考文章：《JavaScript 深入理解之原型与原型链》

七种继承方式

一、ES6继承

二、原型链继承

缺点

  • 多个继承的时候，不能对父类进行修改，否则多个子类会共享一个父类实例（因为所有子类的 prototype 都指向的是父类的实例）
  • 在创建子类型的时候不能向超类型传递参数

三、借用构造函数继承(经典继承方式)

基本思路：在子类构造函数的内部调用父类的构造函数，使用apply() 和 call() 方法以新创建的对象为上下文执行构造函数。

优点：可以在子类构造函数中向父类构造函数传参

缺点

  • 子类只能拿到父类自身的属性和方法，无法拿到原型的方法

四、组合继承(伪经典继承)

基本思路：使用原型链继承原型上的属性和方法；借用构造函数继承实例上的属性。
这样，就在原型上实现了原型属性和方法的复用，也能保证每个实例都拥有自己的属性。

缺点

  • 由于调用2次父类构造函数，一次是在创建子类型的时候，另一次是在子类构造函数内部，子类会包含父类的全部实例属性，但不得不在调用子类从而重写这些属性
  • 造成子类型的原型中多了很多不必要的属性

五、原型式继承

基本思路：基于已有的对象来创建新的对象
实现方式：

  • 在 object 函数内部，通过创建一个临时的实例，并返回，从而实现继承；
  • 也可以通过 Object.create() 代替 object函数
    • 此方法接收两个参数：
      • 作为新对象原型的对象
      • 给新对象定义额外属性的对象（第二个可选）

缺点

  • 原型链继承多个实例的引用类型属性指向相同，存在篡改的可能。
  • 无法传递参数

六、寄生式继承

通过创建一个仅用封装继承过程的函数，通过传入一个对象，然后复制一个对象的副本，然后对象进行扩展，最后返回这个对象。

优点

  • 如果这个对象不是自定义类型时，就是一个简单对象实现了继承

缺点

  • 通过寄生式继承来为对象添加函数，会由于不能做到函数复用，从而降低效率

七、寄生式组合继承

寄生组合式继承是通过借用构造函数来继承属性，通过原型链的混合形式来继承方法

缺点

  • 使用超类型的实例做为子类型的原型，导致添加了不必要的原型属性。
